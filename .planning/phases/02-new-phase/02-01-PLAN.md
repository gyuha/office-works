    record 선언:
      public record MyMenuResponse(Long menuId, String code, String name,
                                   boolean canRead, boolean canWrite)

    canRead, canWrite는 반드시 boolean primitive (소문자) — Boolean 박싱 타입 금지.
    이유: Java record의 accessor는 canRead(), canWrite()로 자동 생성되어 Jackson 직렬화 키가
          "canRead", "canWrite"가 된다. Boolean 박싱으로 선언하면 accessor명이 동일해도
          Jackson이 null 가능성을 고려해 직렬화 방식이 달라질 수 있음.

    클래스 Javadoc 포함:
      "메뉴 접근 권한 응답 DTO. @param menuId 메뉴 PK @param code 메뉴 고유 코드
       @param name 메뉴 표시 이름 @param canRead 읽기 권한 여부 @param canWrite 쓰기 권한 여부"

    import: 없음 (Long, String, boolean은 기본 타입/java.lang).
